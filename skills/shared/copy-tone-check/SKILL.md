# Copy Tone Check

시리즈 캠페인의 **카피·자막·내레이션·CTA 카피들을 한꺼번에 놓고 톤 일관성을 점검**하는 스킬.
편마다 별도로 작성하면 미세하게 톤이 흔들리는 문제를 검출합니다.

## When to use

- 시리즈 모든 편의 카피가 작성된 직후
- 외주에 카피 작업을 분산 발주한 후 통합 검수
- 클라이언트 컨펌 전 마지막 점검
- 시즌 전환·신규 편 추가 시 톤 정합성 확인

**v0.2에서 발견된 갭**: 시리즈 5편의 자막을 따로 보면 좋아 보이지만 함께 놓고 비교하는 단계가 빠졌던 문제 해결.

## 청사진 근거

청사진의 직접 명시는 없으나:
- `claim_evidence_traceability` (시리즈 일관성)
- `series-variation` 스킬의 `tone_floor` / `tone_ceiling` 개념

을 카피 영역에 적용한 우리 시스템 고유 스킬.

## 검출하는 톤 불일치 5가지

### 1. 톤 폭(Tone Range) 일탈

`series-variation` 스킬의 LOCKED 자산에 등록된 `tone_floor`(최저 톤)와 `tone_ceiling`(최고 톤) 범위를 벗어남.

**예시**:
- LOCKED: tone_floor "정중함", tone_ceiling "절제된 자신감"
- V3 카피: "와! 골프 복지가 이렇게 쉬워졌어요!" → tone_floor 미달 (가벼움 초과)
- V5 카피: "FMG가 모든 것을 해결합니다" → tone_ceiling 초과 (과장)

### 2. 호칭·인칭 불일치

같은 시리즈인데 호칭이 흔들림.

**예시**:
- V1: "당신의 회원권" (당신)
- V2: "고객님의 라운딩" (고객님)
- V3: "여러분의 임직원" (여러분)
- V4: 호칭 없음
- V5: "귀사의 임직원" (귀사)

→ 호칭이 다섯 가지. 가장 격식 높은 "귀사"로 통일하거나, "여러분" 같이 자연스럽고 통일된 표현으로 잠금.

### 3. 종결어미 불일치

명령형·서술형·의문형·청유형의 비율이 편마다 흔들림.

**예시**:
- V1: "감으로 고르지 마세요" (명령형)
- V2: "한 라운드를 설계합니다" (서술형)
- V3: "회사 근처에서, 가까이서" (명사 종결)
- V4: "받고 버리지 않는 프리미엄" (수식어 종결)
- V5: "리포트로 확인할까요?" (의문형/청유형)

→ 종결 패턴 5가지. 시리즈 정체성이 흐릿. 2~3가지로 압축 권장.

### 4. 메타포·이미지 어휘 충돌

은유나 비유의 결이 편마다 다름.

**예시**:
- V1: "데이터로 고르세요" (이성·분석)
- V2: "정성을 다합니다" (감성·관계)
- V3: "쉽고 빠르게" (효율·속도)
- V4: "오래 간직되는" (지속·품격)
- V5: "숫자가 증명합니다" (이성·분석)

→ V1/V5는 이성, V2/V4는 감성·정서, V3는 효율. 세 가지 다른 결.
시리즈 정체성이 "신중함"이라면 V3는 어긋남.

### 5. 길이·리듬 편차

자막 글자수가 편마다 차이가 너무 큼 → 시각적 인상이 흔들림.

**예시**:
- V1 핵심 자막: 8글자 + 7글자
- V2: 10글자 + 12글자
- V3: 6글자 + 14글자
- V4: 9글자 + 11글자
- V5: 7글자 + 18글자

→ V5의 18글자는 모바일에서 작아 보임. 시리즈 평균(10글자 전후)과 일관성 깨짐.

## 점검 매트릭스

```
편 │ 자막 1                  │ 자막 2                  │ 종결      │ 호칭   │ 메타포  │ 글자수
──┼─────────────────────────┼─────────────────────────┼──────────┼───────┼────────┼──────
V1 │ 감으로 고르지 마세요     │ 데이터로 고르세요         │ 명령      │ 없음   │ 이성    │ 8/7
V2 │ 한 라운드를 설계합니다   │ 중요한 사람과의 시간을    │ 서술      │ 없음   │ 정성    │ 12/12
V3 │ 회사 근처에서           │ 임직원의 골프 복지가 시작 │ 명사+서술 │ 임직원 │ 효율    │ 6/15
V4 │ 받고 버리지 않는        │ 프리미엄 골프 판촉물      │ 수식      │ 없음   │ 품격    │ 8/10
V5 │ 작게 시작하고           │ 결과는 리포트로 확인합니다 │ 서술      │ 없음   │ 이성    │ 6/14
```

이 표를 만들고 컬럼별로 분포·이탈 자동 검출.

## 점검 알고리즘

### Step 1: 데이터 수집
```
input: 시리즈 전 편의 자막·VO·CTA 카피
parse: 각 카피를 5개 차원으로 분해
output: 위 매트릭스 표
```

### Step 2: 톤 폭 점검
```
process:
  - series-variation의 tone_floor / tone_ceiling 확인
  - 각 카피를 0-10 톤 척도로 자동 분류 (1=가장 가벼움, 10=가장 무거움)
  - floor 미만 / ceiling 초과 검출
output: 이탈 카피 리스트
```

### Step 3: 차원별 분포
```
process:
  - 호칭: 분포표 (3종 이상이면 경고)
  - 종결어미: 분포표 (4종 이상이면 경고)
  - 메타포 카테고리: 분포표 (3종 이상이면 경고)
  - 글자수: 평균·표준편차 (CV 0.4 이상이면 경고)
output: 차원별 위반 항목
```

### Step 4: 권장 수정안
```
process:
  - 가장 자주 등장하는 호칭/종결/메타포로 통일 권장
  - 또는 LOCKED에 등록된 표준 따름
output: 편별 수정 제안
```

## 자동화 vs 사람

| 항목 | 자동 가능 | 사람 필요 |
|------|----------|----------|
| 호칭 분포 | ✅ | — |
| 종결어미 분포 | ✅ | — |
| 글자수 평균·편차 | ✅ | — |
| 톤 폭 점수 | 부분 (LLM 분류) | 미묘한 경우 |
| 메타포 카테고리 분류 | 부분 (LLM 분류) | 100% 사람 권장 |
| 시리즈 정체성 적합성 | — | 100% 사람 |

LLM 분류 결과를 사람이 한 번 확인하는 절차 권장.

## 휴리스틱

- **시리즈 시작 전에 톤 표준 카피 1~2개를 잠금** — 다른 편들이 그걸 참조
- **호칭과 종결은 가장 흔들리기 쉬움** — 외주 작업일수록 더
- **메타포는 시리즈 정체성과 직결** — 한 카테고리 위주(70% 이상) 권장
- **글자수 CV(변동계수)는 0.3 이하 권장** — 시각적 인상이 흔들리지 않음
- **수정안은 2개 이상 제시** — 사용자가 고를 수 있게

## 시스템 호출

- `series-variation`의 LOCKED 자산 참조 (tone_floor / tone_ceiling)
- `guardrail-check` 이전 단계 — 금지 표현이 톤과 무관하게 별도
- `production-brief`의 production_brief.md 최종 작성 전 호출
- `series-closer` 작업 후 (클로저 카피가 시리즈 톤과 어울리는지)

## 오픈크랩 컨텍스트

- 시리즈 톤 분석 결과를 ingest → 차기 시리즈의 톤 기준
- 카테고리별 톤 패턴 누적 (골프회원권은 보통 어떤 톤인지)
